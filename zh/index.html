    开源 · MIT · 自托管

    一个控制台，调度你的整个 Agent 舰队。

    MetaBot 个人版默认运行 Codex，并将 Kimi Code 0.27+ 作为一级引擎——从 Web、飞书/Lark、Telegram 和微信都能访问。Claude Code 保持兼容，登录凭证只留在你自己的机器上。

    安装 MetaBot →查看 GitHub ↗探索 Core Console
    $curl -fsSL https://github.com/xvirobotics/metabot/releases/latest/download/install.sh | bash
    Node.js ≥ 22.19无需 SSO / OIDCToken 权限 0600SHA256 校验
    01 引擎

    默认选择 Codex，Kimi Code 是一级伙伴。

    每个 Bot 都能独立选择引擎与工作区；通道、会话、记忆和监督层保持一致。

    Codex CLI默认
    为编码循环而生。

    codex exec --json 与 resume 通过公开 CLI 提供结构化轮次、工具、模型和推理强度。

    会话 · resume
    控制 · model / effort
    登录 · codex login 或 profile
    Kimi Code 0.27+一级
    本地运行的持久 Agent Server。

    MetaBot 连接 Kimi 官方 loopback Server API——与 Kimi 自己的 Web UI 共用同一套会话。

    会话 · 持久快照
    控制 · 提问 / 停止 / 恢复
    团队 · 子 Agent 状态
    Claude Code兼容
    让已有工作区继续运转。

    现有 Claude Bot、Skills 和会话继续通过兼容 CLI 与 SDK 路径使用。

    后端 · CLI / SDK
    Skills · 保留
    登录 · claude login

    在 bots.json 中按 Bot 配置引擎和工作区。查看多 Bot 配置 →

    02 Core Console

    一个产品，一个 Web UI。

    Core Console Chat 把实时 Agent 执行与完整工作区放进同一套导航。

    01

    看到过程，而不只是答案

    流式回复、工具活动、输出文件、Agent 提问、取消控制、语音输入和持久对话都在同一页。

    02

    管理完整工作区

    Agents、Memory、Skills、T5T、Teams、CLI Access 和诊断共用一个 Token。

    03

    默认只在本机开放

    访问 http://localhost:9200；只有在你主动选择时，才通过反向代理对外暴露。

    Core Console · localhost:9200
    ▮ Chat实时运行与对话
    ▮ Agents引擎与可见性
    ▮ Memory可搜索知识
    ▮ Skills共享能力
    ▮ T5T持久进度
    ▮ Teams任务与运行
    ▮ CLIToken 访问
    ▮ Health运行时诊断
    Token 存放于 ~/.metabot-core/token
    默认文件权限 0600
    03 系统

    远不止一个聊天桥接器。

    这是一个完整的个人 Agent 工作区，每一层都可见、可监督。

    01 / Chat

    实时执行

    跨通道流式展示回复、工具、问题、输出文件和停止控制。

    02 / Agents

    独立 Bot

    每个 Bot 拥有自己的引擎、工作区、凭证、会话和可见性策略。

    03 / Memory

    MetaMemory

    跨运行搜索知识，并可选择同步飞书 Wiki。

    04 / Skills

    共享能力

    将完整 Skill bundle 镜像到 Codex、Kimi、Claude 与 Agent 路径。

    05 / T5T

    持久进度

    记录里程碑、阻塞、验证、发布与最终证据。

    06 / Teams

    并行协作

    通过持久任务和 run 委派工作，Lead 负责最终集成。

    07 / Files

    可见产物

    生成文件与工具活动始终关联到创建它们的运行。

    08 / Sessions

    引擎原生状态

    Codex resume、Kimi 持久会话与子 Agent，以及 Claude 兼容路径。

    04 通道

    在你工作的地方，遇见 Agent。

    浏览器和手机聊天共享同一批 Bot 与会话。

    W

    WebCore Console

    飞

    05 安装

    从零开始，搭好本地 Agent 工作区。

    一个 Release 包安装 Bridge、本地 Core、统一 Web UI、CLI 和内置 Skills。

    terminal · 安装与更新
    # 安装 — 校验 Release checksum
    $ curl -fsSL https://github.com/xvirobotics/metabot/releases/latest/download/install.sh | bash
    # 登录一个或两个引擎
    $ codex login
    $ kimi login
    # 后续更新 — 保留配置与数据
    $ metabot update
    01

    运行 Release 安装器

    解压前校验 SHA256SUMS 与完整个人版 Manifest。

    02

    登录引擎

    使用已有 Codex 或 Kimi Code 订阅登录，凭证不会被复制到网站。

    03

    打开控制台

    访问 http://localhost:9200，使用 ~/.metabot-core/token 中的 Token。

    查看 Linux、macOS 与 Windows 安装详情 →
    06 信任

    你的机器，就是云。

    个人版围绕本地所有权和显式暴露进行设计。

    数据本地状态

    Core 数据位于 ~/.metabot-core，Bridge 会话与用户状态位于 ~/.metabot。

    认证Token 控制台

    本地 Token 以 0600 权限保存，安装器不会打印；也可放在自有代理之后。

    供应链校验更新

    覆盖代码之前，更新器会验证 Release SHA256 与 Manifest 身份。

    IM精确路由

    Allowlist 与精确 @Bot 匹配让多 Bot 群聊行为保持显式。

    网络不强制暴露

    飞书与 Telegram 主动出站连接；本地端口只在认证代理或私网后开放。

    版本为个人而设计

    不依赖企业 SSO、OIDC、VPN、员工目录或任何内部服务。
